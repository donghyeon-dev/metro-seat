# Metro-Seat 로드맵

## 앱 방향성

- **실사용자**: 지하철 이용객 (출퇴근/일상)
- **핵심 가치**: 하차 예정자의 자리를 미리 확보하여 서서 가는 시간을 줄임
- **설계 원칙**: 최소 마찰, 빠른 진입, 조용한 상호작용

---

## 워크플로우 정의

### 자리 제공자 (Provider)

1. 출발역/도착역/탑승 열차 선택
2. 좌석 위치 등록 → **대기 화면** 진입
3. 이용자의 "가고 있어요" 알림 수신
4. 앞에 사람이 서면 → **"앞에 누가 서있어요"** 버튼
5. 이용자가 맞으면 → 간단한 확인 (끄덕임 / 템플릿 메시지)
6. 이용자가 아니면 → **"다른 사람이 앉았어요"** → 자리 자동 취소

### 자리 이용자 (Seeker)

1. 출발역/도착역/탑승 예정 열차 선택
2. 해당 열차에서 이용 가능한 좌석 확인
3. 좌석 선택 → 제공자에게 **"가고 있어요"** 알림 전송
4. 좌석으로 이동
   - 제공자 앞에 아무도 없으면 → 앞에 서고 **"도착했어요"** 알림
   - 제공자 앞에 이미 누가 있으면 → "다른 사람이 있어요" 상태 확인 → 포기
5. 제공자와 간단한 확인 후 자리 이용

## Phase 1: 마찰 제거

> 목표: 앱을 열면 0초 만에 사용 가능

### 1-1. 로그인 벽 제거
- [ ] 앱 최초 접속 시 `signInAnonymously()` 자동 실행 (미들웨어/레이아웃)
- [ ] `/auth/login` 페이지 제거 → 로그인 없이 제공/찾기 즉시 가능
- [ ] 닉네임 설정은 "내 현황" 페이지에서 선택적으로

### 1-2. 제공 플로우 단축
- [ ] 7단계 → 5단계로 축소 (역 선택 시 호선 자동 판별, 확인 단계 간소화)
- [ ] 핵심 정보만: 현재역 → 열차 → 좌석 → 하차역

### 1-3. 홈 화면 컨텍스트 인식
- [ ] 활성 제공/요청이 있으면 홈 상단에 현재 상태 카드 표시
- [ ] "자리 등록 중 → 대기 중 → 매칭됨" 상태 한눈에 확인

### 1-4. UI 다듬기
- [ ] 전체적인 터치 타겟 크기 개선 (지하철 안에서 한 손 조작)
- [ ] 불필요한 텍스트 줄이기, 아이콘 + 색상으로 직관성 확보
- [ ] 로딩/에러 상태 일관성 있게 정리

## Phase 2: 라이브 세션 UX

> 목표: 제공자-이용자 간 실시간 상호작용

### 2-1. 제공자 활성 화면
- [ ] 등록 완료 후 "대기 중" 라이브 화면 진입 (홈에서도 접근 가능)
- [ ] 이용자 요청 도착 시 실시간 알림 (Realtime + 진동)
- [ ] **"앞에 누가 서있어요"** 버튼 → `someone_in_front` 상태 업데이트
- [ ] **"다른 사람이 앉았어요"** → 자리 자동 취소 처리

### 2-2. 이용자 이동 화면
- [ ] 좌석 요청 후 "이동 중" 상태 화면
- [ ] **"도착했어요"** 버튼 → 제공자에게 알림
- [ ] 제공자의 "앞에 누가 있어요" 상태 실시간 확인
- [ ] 이미 다른 사람이 앉은 경우 → "이 자리는 더 이상 이용할 수 없어요" 안내

### 2-3. 간단 확인 수단
- [ ] 매칭된 제공자-이용자 간 템플릿 메시지 (자유 입력 아님)
  - 예: "검은 마스크 쓰고 있어요", "끄덕여 주세요", "감사합니다"
- [ ] 자리 양도 완료 버튼 → 양쪽 모두 "완료" 확인

## Phase 3: 신뢰 & 로그인 유도

> 목표: 자연스럽게 로그인하고 싶게 만들기 (벽이 아닌 경사로)

### 3-1. 신뢰 시그널
- [ ] 비로그인 사용자 = "신규" 배지 (매칭에서 자연스럽게 불리)
- [ ] 로그인 사용자 = 매너 점수 + 이용 횟수 표시
- [ ] 제공자/이용자 프로필에 신뢰도 표시 (좌석 목록에서 보임)

### 3-2. 매너 평가 UI
- [ ] 양도 완료 후 간단 평가 (별점 1~5, 한 줄 코멘트 선택)
- [ ] 매너 점수 "내 현황"에서 확인
- [ ] 신고 UI 연결

### 3-3. 편의 기능 (로그인 시)
- [ ] 자주 가는 경로 저장 (원탭 등록)
- [ ] 즐겨찾기 역
- [ ] 기기 간 프로필 이전 (소셜 로그인: 카카오/구글)

### 3-4. 타이밍 넛지
- [ ] 첫 양도 성공 후: "계정을 만들면 매너 점수가 쌓여요"
- [ ] 3회 사용 후: "경로를 저장하면 더 빠르게 쓸 수 있어요"
- [ ] 감사 표현 수신 시: "확인하려면 계정이 필요해요"
