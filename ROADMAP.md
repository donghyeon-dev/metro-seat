# 자리요 로드맵

> 브랜드명 전환: **메트로시트 → 자리요** (2026-04-17).
> 레포·DB·패키지명 `metro-seat`는 유지, 사용자 대면 문구만 "자리요" 사용.

## 앱 방향성

- **실사용자**: 수도권 1~9호선 출퇴근 이용자
- **핵심 가치**: 하차 예정자의 자리를 **호의로** 이어받아 서서 가는 시간을 줄인다
- **포지셔닝**: "이 앱은 자리를 사고팔지 않습니다. 호의의 교환을 돕습니다."
- **설계 원칙**: 최소 마찰, 빠른 진입, 조용한 상호작용, Provider 부담 최소화

자세한 배경은 `docs/STRATEGY.md` 참조.

---

## 워크플로우 (요약)

### Provider
1. 출발역/도착역/탑승 열차 선택
2. 좌석 위치 등록 → 대기 화면
3. Seeker "가고 있어요" 수신
4. 앞에 사람이 서면 → "앞에 누가 서있어요" 상태 업데이트
5. 확인/자동 취소

### Seeker
1. 출발/도착역 선택 → 열차 선택
2. 좌석맵에서 이용 가능 좌석 선택
3. "가고 있어요" 알림 전송 → 이동 → 매칭 or 포기

## 3개월 실행 로드맵 (Sprint 0 → 4)

> 기존 Phase 1-4는 "기능 중심"이어서 1인 사이드 리소스에 과부하. 아래는 **MVP 증명 중심** 재정렬.

### Sprint 0 — 출시 준비 (1주)
**목표**: 실사용자가 접속할 수 있는 상태로 만든다.

- [ ] Vercel 프로덕션 배포
- [ ] 도메인 연결 (jariyo.app, jariyo.kr 리다이렉트)
- [ ] GA4 측정 ID 환경변수화 + 핵심 5 이벤트 연동
  - `offer_created`, `offer_cancelled`, `request_sent`, `match_succeeded`, `match_failed`
- [ ] Sentry 무료 플랜 연동 (client + server)
- [ ] `/terms`, `/privacy`, `/disclaimer` 페이지 (좌석 판매·거래 금지 명시)
- [ ] robots.txt + sitemap.xml
- [ ] 브랜드 전환: "자리요" 문구·로고·파비콘·manifest 반영

### Sprint 1 — Provider 동기 실험 (2주)
**가설**: 심리적 보상만으로 Provider 재등록율 30%↑ 가능한가?

- [ ] "지금 내리는 중" 홈 플로팅 버튼 (하차 30초 전 원탭)
- [ ] 매칭 성공 애니메이션 + "양보자 뱃지" 카운터
- [ ] Provider 누적 양도 횟수 공개 프로필 (익명)
- [ ] A/B: 뱃지 노출군 vs 비노출군 (GA4 실험)

### Sprint 2 — Seeker 실망 완화 (2주)
**가설**: "가능성" 포지셔닝이 실패 재방문율↑을 만드는가?

- [ ] 좌석맵에 "실시간 매칭 성공률" 표시 (정직한 기대치)
- [ ] 매칭 실패 후 "다음 열차에서 다시" CTA
- [ ] 양방 템플릿 메시지 (자유입력 금지)
- [ ] Seeker 대기화면 "도착했어요" 버튼

### Sprint 3 — 마이크로 커뮤니티 침투 (3주)
**가설**: 2호선 강남↔잠실 오전 7-9시 구간에서 사용자 밀도를 만들 수 있는가?

- [ ] 사용 후기·스크린샷 3개 이상 직접 작성·배포 (에브리타임·블라인드·X)
- [ ] "출근길 양보자" 리더보드 (해당 구간 한정)
- [ ] 매일 본인 실사용 + 더미 Provider 3회 투입 (cold-start 부스팅)
- [ ] 커뮤니티 3곳 침투 계획 실행

### Sprint 4 — 측정·인터뷰·Decision Gate (4주)
- [ ] 누적 매칭 로그 분석 대시보드 (Supabase SQL)
- [ ] Provider 인터뷰 10건 + Seeker 인터뷰 10건
- [ ] **Decision gate**: 주 5건 매칭 달성? Y → Sprint 5~, N → 옵션 C 피벗 검토

## 이번 분기 Non-Goals (의도적 제외)
